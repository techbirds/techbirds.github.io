类(Classes)

核心

从语义上更直观的理解类，这样更加利于类的使用和掌握。ES5的关于类的构建太过于抽象，理解成本比较高，理解的前提是需要弄明白原型链。

  • 构造函数
  • 实例方法
  • 类方法
  • 类继承
  • 实例属性
  • 类属性

ES6

注：

  • 方法之间不需要逗号分隔，加了会报错
  • ES6明确规定，Class内部只有静态方法，没有静态属性。
  • 子类必须在constructor方法中调用super方法，否则新建实例时会报错
  • 类内部方法不需要写function关键字
  • ES6可以自定义原生数据结构（比如Array、String等）的子类，这是ES5无法做到的。
  • extends关键字不仅可以用来继承类，还可以用来继承原生的构造函数。

模块(Module)

核心

javascript终于有了自己的模块化方案了，本质上与社区里的AMD、CommonJS的目标一致。

语法糖

  • 输出模块接口(export)

    • 四种方式

      • 写法一 export var m = 1;
      • 写法二 var m = 1; export {m};
      • 写法三 var n = 1; export {n as m};

      • 写法四(默认输出接口)

        • export default function(){….} // 匿名有效
        • export default function A(){….} // 同样有效的
          注：

      • m或者n可以是基础类型、函数(非匿名)、对象、class等

      • 可支持多个数据类型输出
      • export default只能使用一次

  • 获取模块接口(import)

    • 两种方式

      • 局部加载：import { m } from 'my_module’; // 直接拿m用
      • 全局加载：import * as obj from 'my_module’; // 需要obj.m使用

      • 默认加载(针对export default)：import m from 'my_module';
        注：正因为有默认加载，很多第三方库，可在不知道输出接口名称的前提下直接使用，例如 import _ from ‘lodash’;

        注：

      • 如果仅仅是加载执行某个模块，例如 import 'lodash';

      • 两种情况下，import不需要大括号
        • 全局加载
        • 默认加载

  • 模块可继承

    • 直接在定需要继承的模块里 export * form ‘my_module'

注：

  • 浏览器目前还不支持ES6模块
