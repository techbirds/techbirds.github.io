盗图自阮老师的常用 Git 命令清单

基础

  • 基础工作流
    • 工作目录区
    • 暂存区(Stage)
      • 添加到暂存区 —— git add – file
      • 从暂存区撤回(踢出暂存区) —— git rm – cached file
      • 取消缓存或者重置(跟上者是有区别的) —— git reset HEAD – file
      • 丢弃缓存中修改的内容/所有 —— git checkout – file / git checkout .
    • HEAD区
      • 提交到HEAD区 —— git commit -m [message]
      • 恢复指定文件到暂存区 —— git checkout [sha-1] [file]
    • 远程
      • 推送 —— git push [remote alias] [local branch]
      • 拉取 —— git pull [alias] == (git fetch [alias] + git merge [alias]/[branch])
  • 分支(本地)
    • 新建分支 + 切换分支 —— git checkout -b [new branch]
    • 切换分支 —— git checkout [branch]
    • 合并分支 —— git merge [other branches]
    • 删除分支 —— git branch -d [branch]
  • 其他
    • 冲突解决
    • 标签(给历史记录中的某个重要的一点打上标签)
      • 新建标签 —— git tag -a [标签版本] [sha-1]
      • 删除标签 —— git tag -d [标签版本]
      • 推送标签
        • 单个 —— git push [alias] [tagname]
        • 全部 —— git push [alias] – tags
      • 检出标签 —— git checkout -b [branchname] [tagname]
    • 远端别名 —— git remote add [remote alias] [仓库地址]
    • 撤销操作
      • 重置暂存区与工作区，与上一次commit保持一致 —— git reset –hard
      • 重置当前分支的指针为指定commit，同时重置暂存区，但工作区不变 —— git reset [commit]
      • 重置当前分支的HEAD为指定commit，同时重置暂存区和工作区，与指定commit一致 —— git reset –hard [commit]
      • 重置当前HEAD为指定commit，但保持暂存区和工作区不变 ——git reset –keep [commit]

注：

  • 养成使–作为路径和选项的分隔符的好习惯
  • 拉取动作推荐fetch+merge
  • 关于origin别名生成会在仓库克隆(clone)的情况下自动产生
  • 分支合并时不需要指定被合并的对象，因为默认被合并的对象就是当前所在分支
  • 分支合并之前最好做个对比预览
