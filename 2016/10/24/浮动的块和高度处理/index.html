浮动的块和高度处理

Oct 24, 2016 | CSS | Hits

一、让浮动看起来像普通块

例子

猛戳这里

现象

感觉浮动元素脱离文档流，浮动元素不占文档流，同时下一个兄弟元素受到这种影响，占据了本来应该是浮动元素占据的空间。

Ps. 浮动元素不是真正的脱离文档流，因为被影响的兄弟元素的文字会产生环绕效果。这跟绝对定 位脱离文档流是有本质区别的。

清除

为了清除这种影响，希望达到浮动元素真实占据文档流且兄弟元素占据自己的文档流空间的效果，说白了就是把浮动元素当作正常元素处理。使用clear属性，值根据需求分别left、right和both。

使用

需要小心的是并非是作用到浮动元素，而是作用到被影响元素。

二、让浮动的高度受父元素控制

例子

猛戳这里

DOM

沿用1的DOM

现象

浮动元素高度大于兄弟元素，假设父元素的子元素的所有高度都小于这个高度，那么会产生溢出。

溢出影响

通常我们需要利用浮动元素的高度(最大的时候)，但是因为溢出，显然无法利用这个高度。

清除

为了清除这种影响，即bfc处理便可。

.clearfix { overflow: auto; zoom: 1;// 低版本浏览器添加 }

使用

注意是父容器元素

总结

清除浮动的叫法在理解原理上较为被动，而且很容易混淆，所以重新整理并总结其最为核心的点来加强记忆。
